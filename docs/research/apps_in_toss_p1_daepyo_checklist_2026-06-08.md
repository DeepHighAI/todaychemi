# 앱인토스 출시 — 대표 P1 액션 체크리스트 (외부 선행작업)

- 작성일: 2026-06-08
- 대상: 오늘케미 → 앱인토스 미니앱 출시
- 목적: 미니앱 **개발(P2~P5)을 막는 외부 선행작업**을 대표가 순서대로 실행하기 위한 실행용 문서.
- 근거: 검토 보고서 `apps_in_toss_integration_review_2026-06-07.md`(분석용) → 본 문서(실행용). 가격은 **1,000 / 800 / 600원** 단일(`feature-prices.ts` 정합).
- 표기: 각 항목 = **[무엇을] · [어디서] · [개발 전달분] · [선후/주의]**. 완료 시 `[x]` 체크.

## 0. 한눈에 보기 — 권장 실행 순서

```
① 채널톡 사전 문의 (6번)   ← 가장 먼저! 카테고리 반려 리스크 제거
        │
② 콘솔 가입 + appName 확정 (1번)
        │
③ 사업자/팝빌/세무 (2번) ──┐  (②와 병행 가능)
        │                   │
④ IAP SKU(3) · 토스로그인(4) · mTLS(5)   ← 개발 P3/P5 선결
        │
⑤ 테스트 기기(7)
```

**개발 임계 의존성**: 5(mTLS 인증서) · 3(IAP SKU) · 4(scope·콜백 URL) · 6(채널톡 답변)이 개발의 **P3(인증)·P5(결제)** 선결 조건. 1(appName)은 **P2 라우트 매핑**에 필요. 6-⑦(계정 링크)은 **계정 모델 결정**에 직접 영향.

## 1. 콘솔 가입 → 워크스페이스 → 앱 등록

- [ ] **무엇을**: 앱인토스 개발자센터 가입 → 워크스페이스 생성 → 앱 등록.
- [ ] **appName 확정** (예: `twoday` 류 영문 slug). ⚠️ **사실상 영구 고정** — 딥링크(`intoss://{appName}/...`)·서빙 도메인(`{appName}.apps.tossmini.com`)에 영구 사용. 오타·변경 시 신규 앱 재등록 필요. 정책위반/중복 지적 시에만 최초 24h 내 1회 변경 가능.
- [ ] 로고, 사용 연령, 고객센터 이메일/연락처 등록.
- [ ] **앱 내 기능 최소 1개 등록**(비게임 필수): 기능명 + `intoss://{appName}/경로`. 토스 UX 라이팅(’~하기’/명사형). 검토 1~2영업일.
- **어디서**: https://apps-in-toss.toss.im
- **개발 전달분**: 확정된 **appName**, 등록한 **기능명↔경로 매핑**.
- **선후/주의**: 카테고리는 **‘라이프스타일/운세’ 계열로 등록**, ‘소셜/만남’ 회피(데이팅 규제 회피 — 오늘케미는 이용자 간 매칭 없음). 채널톡(6번) 답변 확인 후 등록 권장.

## 2. 사업자 / 정산 인프라 (세무사 상담 포함)

- [ ] **무엇을**: 개인/법인 사업자 서류 제출. (사업자 없이도 출시는 가능하나 **유료 결제·정산엔 사업자 필요**.)
- [ ] **팝빌(Popbill) 가입** + **사업자 공동인증서**(연 4,400원) 등록.
- [ ] **세무사 상담**: 웹(토스페이먼츠)·미니앱(IAP) **이원 정산** 구조의 세무 처리(단일 사업자 가능 여부, 부가세/원천 등).
- **어디서**: 홈택스/은행(인증서), 팝빌 사이트, 세무사.
- **개발 전달분**: 없음(정산 인프라). 단 정산 가능 시점이 유료 출시 가능 시점.
- **선후/주의**: 정산은 **역발행 승인 방식** — 미승인 2회 이상 시 **미니앱 운영 중단** 가능. 익월 말 지급.

## 3. IAP 상품 3종 등록

- [ ] **무엇을**: 콘솔에 인앱결제(IAP) 상품(sku) 3종 등록.
  - [ ] 케미카드(합카드) — **1,000원**
  - [ ] 만약에 우리(만약합) — **800원**
  - [ ] 케미 다시 맞추기(다시합) — **600원**
- [ ] 각 상품명·아이콘·설명 작성. **VAT 포함가 기준** 확인.
- **어디서**: 앱인토스 콘솔(인앱결제).
- **개발 전달분**: 확정된 **3개 sku ID**(개발이 `getProductItemList`/`createOneTimePurchaseOrder`에 사용).
- **선후/주의**: 가격은 코드 단일출처(`feature-prices.ts` 1,000/800/600)와 **반드시 일치**. 변경 시 §1.1 + ADR-039 동시 갱신.

## 4. 토스 로그인 콘솔 설정

- [ ] **무엇을**: 토스 로그인 약관 등록 + scope 선택 + 콜백 설정.
- [ ] **scope = `user_key` only** (D2 확정 — PII 무수집, 복호화 키 불필요). birthday/gender 등 PII scope **미선택**.
- [ ] **연결끊기(탈퇴) 콜백 URL** 등록 → 개발이 받을 엔드포인트(`/api/toss/disconnect`).
- **어디서**: 앱인토스 콘솔(로그인).
- **개발 전달분**: 토스 로그인 **client 설정값**, **콜백 URL**(개발과 협의해 확정).
- **선후/주의**: scope를 user_key only로 하면 온보딩(생년월일·시·성별)은 **사용자가 직접 입력**(토스 PII 아님)하므로 문제 없음. 추후 email-scope 링크 가능 여부는 채널톡 6-⑦에서 확인.

## 5. mTLS 인증서 발급

- [ ] **무엇을**: 파트너 서버 통신용 mTLS(양방향 TLS) **클라이언트 인증서 발급**.
- [ ] 무중단 교체 대비 **2개 이상** 발급 권장.
- **어디서**: 앱인토스 콘솔(서버 연동/인증서).
- **개발 전달분**: **cert/key(PEM) 파일** → 개발이 Vercel 환경변수(base64)로 보관. **시크릿이므로 안전 채널로 전달**(이메일·메신저 평문 금지).
- **선후/주의**: 토스 로그인 토큰 교환·IAP 주문 검증·탈퇴 콜백 전부 mTLS 필수. **개발 P3 착수 선결**.

## 6. 채널톡 사전 문의 8건 (⭐ 가장 먼저 권장)

- [ ] **무엇을**: 콘솔 채널톡으로 아래 8건 문의 → 답변을 개발에 전달(반려·재작업 리스크 사전 제거).
- **어디서**: 앱인토스 콘솔 채널톡.
- **개발 전달분**: 각 문항 **답변 원문**(특히 ②③④⑥⑦은 설계 분기).

> **복붙용 문의안**:
> 1. **(카테고리)** 오늘케미는 관계 사주 해석(케미카드)·일일 운세(오늘 케미)·가정 시뮬레이션(만약에 우리) 서비스입니다. 이용자 간 매칭·채팅이 없고 인연은 사용자가 입력하는 비공개 데이터입니다. ① 데이팅/소개팅 규제 대상인가요? ② **돈합(재물운) 모드**가 투자자문/금융상품 추천으로 분류될 위험이 있나요? ③ 권장 등록 카테고리는?
> 2. **(TDS)** 비게임 WebView 미니앱에서 **TDS(@toss/tds-mobile) 사용이 검수 필수**인가요, 권장인가요? 필수라면 적용 범위(셸만 vs 콘텐츠 영역 포함)는?
> 3. **(부적/가상통화)** 무료로 매일 적립되는 인앱 크레딧(‘부적’)으로 기능을 잠금해제하고, 부족 시 IAP로 결제하는 **하이브리드** 구조가 허용되나요? 무료 크레딧이 ‘가상통화’로 제한되나요?
> 4. **(익명키)** 비로그인(게스트) 식별용 `getAnonymousKey` 같은 API가 SDK 2.6.x에 있나요? 있다면 시그니처/제약은?
> 5. **(에러리포팅)** WebView 미니앱에서 **Sentry** 등 외부 에러 리포팅 SDK가 동작하나요? 권장 관측성 메커니즘은?
> 6. **(IAP 지급 SLA)** `processProductGrant` 30초 한도와 관련해, 서버 `order/get-order-status` 검증의 **권장 지연 한도/타임아웃**은?
> 7. **(계정 링크)** `user_key`만으로 로그인한 사용자가 추후 **email scope**를 받아 기존(웹) 계정과 연결할 수 있나요?
> 8. **(임베드)** iframe 제한의 정확한 범위와 **YouTube 등 외부 임베드 허용** 여부는?

## 7. 테스트 기기 준비

- [ ] **무엇을**: 샌드박스 앱 설치용 실기기 확보(**iOS 16+ / Android 7+**).
- **어디서**: 보유 기기 또는 신규.
- **개발 전달분**: 없음(QA·검수 단계 사용).
- **선후/주의**: TDS 컴포넌트·분석/공유리워드/인앱광고는 **로컬 브라우저 미지원** → 샌드박스/QR 토스앱에서만 검증 가능. 검수 요청은 **테스트 1회 이상 완료** 후 활성화.

## 완료 게이트 (개발 P2~P5 진입 조건)

| 개발 단계 | 선결 항목 |
|---|---|
| **P2 미니앱 셸** | 1 appName 확정 |
| **P3 인증 브릿지** | 5 mTLS 인증서 · 4 scope/콜백 URL · 6-④⑤⑦ 답변 |
| **P5 IAP 결제** | 3 IAP SKU 3종 · 5 mTLS · 6-③⑥ 답변 |
| **카테고리/설계 확정** | 6-①②(카테고리) · 6-②(TDS) |

> 본 체크리스트의 모든 항목이 완료되면 개발이 P2부터 순차 진입한다. 진행 중 ‘계정 모델(웹↔미니앱 분리/링크)’ 결정은 6-⑦ 답변 + §1.1 사용자 확정으로 P3 진입 전 마무리한다.
