# Design: Flow A 절충안 + 진단 폼 폐기 (2026-06-13 사용자 확정)

> 잔여 개발 계획 수립(기획서·PRD·UI/UX 전수 대조) 중 발견된 기획-구현 괴리 2건의 §1.1 결정 기록.
> 브레인스토밍 결정: D3(절충) · D4(오늘홈 hero 강화) · D5(진단 폼 폐기) · D6(설계 승인).

## 배경

| 괴리 | 기획 원안 | 현 구현 |
|---|---|---|
| Flow A (ADR-032·029) | 가입 직후 인연 등록 입구(S-01-A) 우선, 본인 정보는 인연 등록 흐름 안 인라인, 생일 unknown → Track B "첫만남 플레이" 분기 | 본인 4-step 온보딩(생일→시간→달력·성별→확인) → 오늘 홈. 인연 등록은 별도 진입. Track B 분기 없음 |
| S-06 진단 폼 (FGI §3.2) | 모드 선택 + 5-7문항 응답 → `question_slot`으로 LLM 전달 | 모드 1탭 선택만. `question_slot` 배선은 존재하나 항상 빈 값 |

## 결정 1 — Flow A 절충안 (D3·D4)

**원칙**: 현 본인-first 온보딩은 유지한다. 본인 정보만으로도 즉시 가치(오늘 케미·본명식)를 주고 있고, 여러 차례 §1.1 승인을 거쳐 라이브 검증된 흐름이다. 부족한 것은 온보딩 이후 인연 등록까지 끌고 가는 힘이며, 이를 **전용 화면 추가 없이** 기존 표면에서 보강한다.

### 변경 사항
1. **오늘홈 인연 0건 hero 재구성** — 인연이 0건일 때 오늘홈 hero를 인연 등록 유도 중심으로 강화:
   - ADR-032 카피 의무를 hero에서 수행: "오늘 떠오르는 사람부터 등록해보세요" + 본인 정보 사용 목적 압축 카피
   - 인연 등록 CTA를 hero의 주역으로
   - 기존 환영 팝업(G1)은 서비스 소개, hero는 행동 유도로 역할 분담
2. **Track B 분기 현실화 (ADR-029)** — `/relations/new` 생일 단계에 "생일을 잘 몰라요" 선택지 추가:
   - 선택 시 "처음 보는 나"(whatif `first` — 기구현 콘텐츠)로 안내하는 분기 카드 노출
   - 등록 강제 중단이 아니라 대안 경로 제시 (등록 계속 가능)

### 채택하지 않은 안
- 기획 원안 구현(인연-first 온보딩 재정렬): 검증된 UX 재작업 + 본인 정보 인라인 입력 UX 난도 상승 — 기각(D3)
- 전용 입구 화면 1장(S-01-A interstitial): 퍼널 길이 +1 — 기각(D4, hero 강화 선택)

## 결정 2 — S-06 진단 폼 폐기 (D5)

**5-7문항 진단 폼은 영구 미도입.** 경량 1문항 안도 기각. 근거:
- 결제 전 마찰 최소화 우선
- 해석 개인화는 파생층·교차분석(ADR-040)이 이미 담당 — 십신 교차·궁위·운세 교차로 "내 상황" 밀도 확보

### 코드 처분
- `question_slot` 필드(API·LLM 페이로드·타입)는 **예약(reserved) 필드로 유지** — 제거 시 타입 ripple만 크고 무해함. 신규 코드가 채워 넣지 않는다.
